# 🚢 Titanic Survival Prediction with Deep Learning

一个使用 **PyTorch** 构建的深度学习模型，用于预测泰坦尼克号乘客的生存情况。本项目并非简单的入门练习，而是包含了**独特的特征工程、动态训练策略和严谨的模型评估**，在Kaggle竞赛中0.9425 的成绩。

## 📊 项目概述

本项目目标是构建一个二分类模型，根据乘客的票务、姓名、舱位等特征，预测其在泰坦尼克号海难中的生存(`Survived=1`)或遇难(`Survived=0`)。

**核心成果**:
*   **模型性能**：在保留的验证集上取得了 **85.5%** 的准确率和 **0.836** 的F1分数。
*   **技术实践**：完整实现了数据清洗、特征工程、神经网络构建、动态训练调优及结果分析的全流程。

## ✨ 关键特性与创新点

### 1. 深入的业务导向型特征工程
*   **真实票价计算**：根据`Ticket`字段，识别同行组，将总票价`Fare`均摊，更准确地反映个人支付能力。
*   **精细化年龄填补**：利用`Name`中的称呼(`Mr.`, `Mrs.`, `Miss`, `Master`)，基于社会角色设定不同默认年龄（如Mr.设为30岁，Master设为16岁），而非简单使用整体均值/中位数。
*   **家庭成员逻辑调整**：将`SibSp`(兄弟姐妹/配偶)和`Parch`(父母/子女)为零的乘客数量设为1，表示“乘客本人”，使特征更符合逻辑。
*   **船舱等级编码**：提取`Cabin`首字母并转换为有序数字特征，捕捉舱位位置与等级信息。

### 2. 精心设计的神经网络与训练策略
*   **动态优化器切换**：在训练后期（损失稳定后），自动将优化器从 **Adam** 切换为 **SGD**，并显著降低学习率(`1e-3` -> `1e-5`)，以实现更精细的参数收敛，防止震荡。
*   **自适应学习率调整**：实现了“耐心”计数器，仅在模型性能停滞时才触发优化策略变更，体现了对训练过程的深度监控。
*   **健壮的模型结构**：网络简洁，集成了`ELU`激活函数、`BatchNorm1d`批归一化层，并预留了`Dropout`层接口以应对过拟合。
*   **全面的评估体系**：除准确率外，持续跟踪并可视化**精确度、召回率、F1分数**，多维度评估模型性能。

### 3. 严谨的模型验证与分析
*   **多维度测试**：除了标准验证集，额外构建了“全女性存活/全男性死亡”的极端案例测试集和“全男性”样本集，深入验证模型在不同子群体上的表现与偏差。
*   **完整流程闭环**：代码包含最终对Kaggle测试集的预测与结果文件(`结果.csv`)生成功能。
*   **模型持久化**：训练完成后自动保存最优模型参数(`模型.pth`)。

## 🛠️ 技术栈

*   **框架**：PyTorch
*   **编程语言**：Python
*   **数据处理**：Pandas, NumPy, Scikit-learn (`StandardScaler`, `train_test_split`)
*   **可视化**：Matplotlib
*   **评估指标**：Accuracy, Precision, Recall, F1-Score (from `sklearn.metrics`)

## 🚀 快速开始

### 环境配置
1.  确保已安装 Python 3.7+。
2.  安装依赖包：
    ```bash
    pip install torch pandas numpy scikit-learn matplotlib
    ```

### 运行步骤
1.  **获取数据**：从 [Kaggle Titanic 竞赛页面](https://www.kaggle.com/c/titanic/data) 下载 `train.csv` 和 `test.csv`，放置于 `data/` 目录下。
2.  **运行代码**：直接运行 `titanic-deeplearning.ipynb` 中的所有单元格。
3.  **查看结果**：预测结果将保存在 `outputs/结果.csv` 中，可用于提交至Kaggle。模型权重将保存在 `models/模型.pth`。

## 📈 结果与分析

训练过程稳定，损失持续下降，未见明显过拟合。模型在复杂场景下表现出色：
*   **验证集**：准确率 **85.5%**, F1分数 **0.836**。
*   **“女性全存/男性全亡”测试集**：准确率 **87.6%**，证明模型成功捕捉到了数据中“妇孺优先”的强规则。
*   **“全男性”子集**：准确率约 **82.4%**，说明模型在去除了最强特征（性别）后，依然能依靠其他特征（舱位、票价等）进行有效预测。

详细的损失及指标变化曲线可在代码运行后查看。

## 💡 未来改进建议

1.  **特征工程**：可尝试提取`Name`中的姓氏以更精确识别家庭；对`Age`进行分箱或构建多项式特征。
2.  **模型架构**：尝试更深的网络，或使用树模型（如LightGBM）进行特征筛选与模型融合。
3.  **超参数调优**：使用`Optuna`或`GridSearchCV`对网络层数、神经元数量、学习率计划进行系统调优。
4.  **交叉验证**：采用K折交叉验证以获得更稳健的性能估计。
